# target-prompt

你是一个异构信息网络的社区搜索算法专家。我现在有一个基于 **Star-schema** 的维护课题，请你深入分析该课题，并依照下面给出的研究思路，补充完整的解决路径与详细说明。

## 一、研究背景

基于星形异构信息网络（Star-schema HINs）的凝聚社区搜索问题中，诸如 `(k, P)-core` 等相关内聚模型，目前仍缺乏有效的**动态维护算法**研究。也就是说，当 Star-schema HINs 发生动态变化时，现有内聚模型难以高效地保持有效性，需要重新设计对应的维护方法。

## 二、核心挑战

请重点围绕以下几个问题展开分析：

1. **如何定义 Star-schema 的原子更新。**
2. **如何针对该原子更新进行高效的 `(k, P)-core` 维护。**
3. **如何在维护过程中避免各元路径对应导出子图上的冗余计算。**
4. **如何维护有效的索引结构。**

## 三、问题建模

本课题采用 **CM-tree** 的压缩存储方式进行建模。

我们首先将 Star-schema 上的一次**原子更新**定义为：

> 一个 `center node` 类型节点的插入 / 删除，以及该 `center node` 到各个 `attribute node` 类型节点之间边的插入 / 删除。

该原子更新会带来以下影响：

1. 在**相同元路径**对应的导出子图中，可能新增或删除多条边。
2. 在**不同元路径**对应的导出子图中，也可能引发边的新增或删除。

因此，整个维护过程可以拆分为两个关键步骤：

### Step 1：提取 path instance 的变化

识别该原子更新对所有 `path instance` 带来的影响，即找出所有新增或减少的 `path instance`。

### Step 2：维护 CM-tree 的有效性

基于上述 `path instance` 的变化，对 CM-tree 结构进行增量维护，保证其在动态更新后的正确性与有效性。

## 四、候选方案

### Baseline 1：全量重算

针对变化后的 HIN，直接对 CM-tree 进行全量重构。

### Baseline 2：Naive Maintenance（DFS + Order-based Decomposition）

基本思路如下：

1. 通过 DFS 找出 CM-tree 中所有节点对应元路径上的 `path instance` 变化。
2. 为每个节点单独维护一个 `order-based` 序列。
3. 对每个节点对应的元路径执行 `order-based` 维护。
4. 将维护结果写回 CM-tree。

可进一步思考的问题：

- 是否需要存储全部导出子图？
- 若不全部存储，如何在时间与空间之间做权衡？

### Advanced Maintenance

在 Naive Maintenance 的基础上，进一步利用 **CM-tree 的嵌套结构**，识别并消除以下两类冗余：

1. `path instance` 搜索阶段中的冗余计算。
2. `order-based maintenance` 阶段中的冗余计算。

最终目标是通过系统性的剪枝策略，提升整体维护效率。

## 五、输出要求

请基于上述背景与思路，输出一份系统化分析，至少包含以下内容：

1. 对问题的形式化定义。
2. 对原子更新模型的进一步细化。
3. 针对 `(k, P)-core` 维护的可行算法框架。
4. 针对多元路径导出子图冗余维护的优化策略。
5. CM-tree 索引维护机制设计。
6. Baseline 与 Advanced 方案的详细流程、关键数据结构、复杂度分析以及适用场景对比。
7. 可进一步展开的理论问题、实验设计建议与潜在创新点。
